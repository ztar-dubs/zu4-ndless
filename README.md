# zu4-ndless

A port of **Ultima IV: Quest of the Avatar** to the **TI-Nspire CX CAS** calculator, based on [zu4 (Zesty Ultima IV)](https://github.com/0ldsk00l/zu4).

## Requirements

- **TI-Nspire CX** or **CX CAS** (color screen required)
- [Ndless](https://ndless.me/) jailbreak installed
- **Build tools:** [Ndless SDK](https://github.com/ndless-nspire/Ndless) with `nspire-gcc`, `nspire-g++`, `genzehn`, `make-prg`
- **Build environment:** Cygwin (Windows) or Linux with the Ndless toolchain in `$PATH`

## Building

```bash
# Ensure Ndless SDK tools are in PATH
export PATH="/ndless-sdk/bin:/ndless-sdk/toolchain/install/bin:$PATH"

# Build
make clean
make

# Output: ultima4.tns (auto-copied to prod/)
```

The build produces `ultima4.tns` (~768 KB) and copies it to `prod/`.

### Build notes

- **Do NOT use `genzehn --compress`** — it causes image loading failures at runtime.
- The linker flag `--wrap=fopen` intercepts all `fopen()` calls to handle the `.tns` extension requirement (see `nspire/nspire_fopen.c`).
- LTO (`-flto`) and `-Os` are used to minimize binary size.

## Architecture

This port replaces several components of zu4 to work within the constraints of the TI-Nspire:

| Original (zu4)  | Replacement (nspire/)        | Why                                      |
|------------------|------------------------------|------------------------------------------|
| SDL2             | SDL1 (nSDL)                  | nSDL is the only SDL available on Nspire  |
| OpenGL renderer  | Software RGBA32→RGB565 blit  | No GPU on Nspire                          |
| libxml2          | yxml SAX parser + DOM shim   | libxml2 too large for calculator          |
| SDL audio        | Audio stubs (no-ops)         | No audio hardware                         |
| SDL threads      | Polling timers (SDL_GetTicks) | nSDL has no thread support               |
| Standard fopen   | Linker --wrap=fopen          | All Nspire files require .tns extension   |

### Key design decisions

- **Screen:** zu4 renders at 320x200, centered on the 320x240 Nspire display (20px black borders top/bottom).
- **Shift = Alt:** The calculator has no Alt key, so Shift is mapped to Alt for command combos (Shift+H for help, etc.).
- **ON+ESC:** Force quit combo (standard Ndless convention), instead of just ESC which could be pressed accidentally.
- **Non-fatal assertions:** `zu4_assert` does not call `abort()` on NSPIRE — out-of-range values are silently clamped.
- **Text node support in XML shim:** The yxml-based XML shim handles `YXML_CONTENT` to parse text nodes, which is required for `vendorScript.xml` (shop dialogues).

## Modifications to zu4

The following zu4 source files have been modified for the Nspire port (all changes are guarded by `#ifdef NSPIRE`):

| File              | Changes                                                    |
|-------------------|------------------------------------------------------------|
| `error.c`         | `zu4_assert` non-fatal (no `abort()`)                     |
| `event.h`         | Polling timer fields (`lastTick`, `running`, `poll()`)     |
| `game.cpp`        | Shift labels in help, sound keys disabled, ESC quit        |
| `intro.cpp`       | `bSkipTitles=true`, ESC as quit key from menu              |
| `image.c`         | Null pointer protection in get/set_pixel, fill             |
| `imageview.cpp`   | Null image checks, silent return on NSPIRE                 |
| `screen.cpp`      | Null charsetInfo protection, tileanimSets cleanup          |
| `textview.cpp`    | Range-check clamp instead of assert                        |
| `tileview.cpp`    | Range-check clamp instead of assert                        |
| `u4file.c`        | `zippath`/`dospath` point to `data/` on NSPIRE             |
| `utils.h`         | `long&` AdjustValue overloads for ARM `int32_t`            |
| `config.cpp/.h`   | Added `destroy()` for clean memory release                 |
| `creature.cpp/.h` | Added `destroy()` + destructor for clean memory release    |
| `imagemgr.cpp/.h` | `screenInfo` tracked as member for proper cleanup          |

## Installation on Calculator

See `prod/doc/readme.html` (English) or `prod/doc/lisezmoi.html` (French) for detailed installation instructions.

In short: copy the entire `prod/` directory contents to `/documents/ultima4/` on your calculator, preserving the directory structure.
